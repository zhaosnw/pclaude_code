# Alignment Status - 2026-07-15

## 决策：对齐目标从 recovered 快照改为正式版

此前 oracle 钉定在 `2.1.87`（npm 上离 recovered 快照 2.1.88 最近的可用版本）。在对 `--max-turns` 做差分时发现 **2.1.87 本身有 bug**：

| 来源 | `error_max_turns` 的 `is_error` | 退出码 |
|---|---|---|
| 2.1.87（旧 oracle） | `false` | 0 |
| 2.1.88 recovered 源码 | `true`（`QueryEngine.ts:856`） | — |
| 2.1.209（正式版） | `true` | 1 |
| hare | `true` | 1 |

2.1.87 是唯一异类，2.1.88 已修。**用户决定：对齐目标改为正式版**（recovered 只是恰好能拿到的快照，正式版才是真正的目标）。

- 新 oracle：`.oracle/claude-2.1.209/`（从本机安装的 `claude` 复制钉入，`.gitignore` 覆盖，459M 不入库）。
  - 注意：2.1.207+ 的 npm 包已改成原生二进制启动器（无 `cli.js`），不能像 2.1.87 那样直接 `node cli.js`；改为复制已安装的二进制。
  - 本机 `claude` 会自动更新（录制期间从 207 跳到 209），故复制进仓库钉住。
- 录制入口：`CLAUDE_TS_CLI="$PWD/.oracle/claude-2.1.209/bin/claude.exe" python scripts/record_golden.py <case_id>`
- 旧 `.oracle/claude-code-2.1.87/` 已删除。

## 重录全部 golden 后暴露的正式版行为差异

用 2.1.209 重录 62 个 case，逐一对齐。除 schema 漂移外，抓到 **7 处真实行为差异**（多为正式版相对 2.1.87 的行为演进）：

1. **`ttft_stream_ms` 新字段**：正式版结果对象新增，hare 缺失。补 `hare/main.py`。（这是大多数重录失败的原因，纯 SDK 契约。）

2. **Edit 不再要求先 Read**：2.1.87 拒绝编辑未读文件（"File has not been read yet."），**正式版直接编辑成功**。移除 `hare/tools_impl/FileEditTool/file_edit_tool.py` 的硬门，read-state 仅保留用于 staleness 检查。

3. **hook 拦截现在计入 `permission_denials`**：2.1.87 不计入（当日早前还专门记过这条"反直觉行为"），正式版改了。hook 拦截路径现在把决策过一遍 canUseTool wrapper 以登记 denial。

4. **坏的 `--mcp-config` 不再中止**：2.1.87 退出 1，**正式版静默跳过坏 server、退出 0、无 stderr**。`hare/main.py` 改为记 debug 日志而非 `sys.exit(1)`。

5. **print 模式不做 auto-compact**：2.1.209 下即使真实 300k-token 会话 + `CLAUDE_AUTOCOMPACT_PCT_OVERRIDE=1` 仍不压缩（`-p` 是单次运行，压缩是 REPL/resume 的事）。`auto_compact_if_needed` 在 `is_non_interactive_session` 时直接返回。PreCompact/PostCompact 从 P1 降为 P2。

6. **`session.resume_bad_id` 分歧消解**：这条一直挂着的 xfail——2.1.87 对未知 session id 静默 exit 0，hare 坚持 exit 1 报错。**正式版和 hare 一致**（exit 1、stderr 报错）。顺带修 hare：错误从 stdout 挪到 stderr、退出码从 0 改 1（原先既污染 json 输出又谎报成功）。

7. **`-p` 值可选**：`echo "prompt" | claude -p` 是标准管道用法，hare 之前用 argparse 强制 `-p` 带值而报错退出。改 `nargs="?"`。

## 新增 CLI flag 支持 + golden

- **`--settings`**（`main.tsx:521`）：额外 settings 文件，叠加在设置链之上（permission 规则 + hooks 都认）。新 case `cli.settings_flag`。
- **`-p` 管道用法**：新 case `cli.print_stdin`（`-p` 无值 + stdin prompt）。

## Subagent 在 2.1.209 变成异步（未对齐，待办）

探针显示正式版的 Agent 工具是**异步**的：`Task` 立即返回 "Async agent launched..."，结果之后经 TaskOutput 取。hare 是同步子代理。`subagent.task_dispatch` 的 golden 仍用旧同步语义录制并通过——但这是**架构级差异**，标记为下一轮待办。（这轮该 case 用 2.1.87 时的行为，2.1.209 录制会挂起，因位置式 fixture 与异步调用序列不匹配。）
