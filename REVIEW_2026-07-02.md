## 当前主要问题

### 1. 目录级资产重复仍未彻底收口

- 仓库当前仍并存 `hare/alignment/` 与 `legacy_alignment/` 两层对齐资产目录。
- 当前不再把 `hare/` 子项目目录视角当作受支持执行入口；根目录是唯一执行入口。

### 2. `mypy` 回归门已恢复，但类型债仍重

- 当前 canonical 扫描范围已排除 `hare/tests/`、前端、文档和脚本目录；`hare/hare/` 已被完全移除。
- 当前基线已更新为 `520`，`make mypy-regression` 与 `make -C hare mypy-regression` 都能通过并阻止新增回归。
- `make typecheck-full` 仍会展示完整的 `520` 个既有类型错误，后续需要按模块继续清偿。

### 3. 真差分覆盖仍然偏薄

- 当前通过的重点还是主链路 E2E，不等于更大范围行为已经完成对齐。
- 各轴现状（详见 `docs/alignment-status/parity-matrix.md`）：
  - `--resume` / `--continue`：✅ 已有 golden（2 项 `aligned` + 1 个已登记 xfail 分歧）
  - permission modes × settings：Python 侧已实现并有 request-side 单测；TS golden 因 oracle 挂起未录
  - hooks：runtime 管线已实现（`tests/test_tool_hooks_runtime.py`）；TS golden 未录
  - MCP：stdio 接线完成 + 1 个 invalid-config golden；tool call golden 因 oracle 挂起未录
  - subagent / Agent 嵌套执行：未开始
  - compact / auto-compact：未开始（待先调查 print-mode 触发方式）
  - transcript JSONL schema：部分覆盖（roundtrip 测试），无专门 golden
- **当前系统性阻塞**：recovered TS CLI 在录制"授权工具调用 / MCP tool call"类 case 时挂起。
